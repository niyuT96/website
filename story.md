# Scrum Story Summary

## Goal
Deliver a minimal-tech personal portfolio site with sections for My Projects, Work Experiences, Education Experience, Skills, and Kontakts, styled after the provided reference image.

## Users
- Primary: recruiters and potential clients evaluating work and experience.
- Secondary: collaborators browsing design and engineering skills.

## Story Count
8 stories in the initial sprint plan.

## Stories
1) As a visitor, I can quickly understand who the designer is from the hero section so I can decide to explore further.
   - Acceptance: name, role, short intro, and clear call-to-action buttons are visible on load.
   - Status: Done (`index.html` hero section).
2) As a visitor, I can browse a curated list of projects so I can assess the creator's capabilities.
   - Acceptance: at least three project cards with title, short description, and link call-to-action.
   - Status: Done (`index.html` #projects).
3) As a visitor, I can review work experience so I can see career progression.
   - Acceptance: timeline list with roles, companies, and dates.
   - Status: Done (`index.html` #experience).
4) As a visitor, I can view education experience so I can gauge academic background.
   - Acceptance: education items include degree, school, and years.
   - Status: Done (`index.html` #education).
5) As a visitor, I can review skills grouped by category so I can understand strengths.
   - Acceptance: skills grouped into design, engineering, and tools.
   - Status: Done (`index.html` #skills).
6) As a visitor, I can reach out via a contact area so I can start a conversation.
   - Acceptance: contact form fields and contact details are visible.
   - Status: Done (`index.html` #contact).
7) As a visitor, I experience a modern, minimal-tech visual style so the site feels polished and contemporary.
   - Acceptance: gradient background, soft cards, clear typography, and subtle motion.
   - Status: Done (`styles.css` base theme + animations).
8) As a visitor on mobile, I can access all sections comfortably so the site is usable on small screens.
   - Acceptance: layout stacks, readable typography, and tap targets remain accessible.
   - Status: Done (`styles.css` responsive breakpoints).
